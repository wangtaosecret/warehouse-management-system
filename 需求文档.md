# 仓库管理系统需求文档

> 版本：v1.0  
> 日期：2026-04-09  
> 状态：初稿

---

## 一、项目概述

### 1.1 项目背景
构建一套支持**多组织隔离**、**多角色权限管控**、**全流程仓储管理**的综合性仓库管理系统，参考成熟开源仓储系统设计，确保系统实用性、安全性和可扩展性。

### 1.2 核心目标
- 实现多组织数据完全隔离，各组织独立开展仓储业务
- 三级权限架构（系统管理员 → 组织管理员 → 组织员工）
- 覆盖商品管理、供应商管理、出入库管理、库存管理全流程
- 补充盘点管理、库存预警、报表统计、日志管理等拓展功能

### 1.3 术语表

| 术语 | 说明 |
|------|------|
| 组织 | 系统下的独立租户，拥有唯一组织ID，数据完全隔离 |
| 货位 | 仓库内具体存储位置，支持多级（区域→货架→货位） |
| 盘盈/盘亏 | 实际盘点数量与系统库存的差异 |
| 单据状态 | 申请 → 审核 → 执行 → 已完成/已归档 |

## 二、角色与权限体系

### 2.1 三级权限架构

```
系统管理员（最高权限）
    └── 组织管理员（单组织最高权限）
            └── 组织员工（自定义角色）
```

### 2.2 系统管理员
| 权限类别 | 具体内容 |
|----------|----------|
| 系统配置 | 全局参数、全局字典、备份恢复 |
| 组织管理 | 创建/编辑/禁用/删除组织，分配组织管理员 |
| 账号管理 | 管理所有组织管理员账号（不可操作组织员工） |
| 日志管理 | 查看系统级日志、所有组织操作日志汇总 |
| 数据管理 | 系统级备份恢复、查看数据占用、清理过期日志 |

### 2.3 组织管理员
| 权限类别 | 具体内容 |
|----------|----------|
| 组织配置 | 本组织基础参数、仓库信息、预警阈值 |
| 员工管理 | 创建/编辑/禁用/删除员工账号，分配角色 |
| 权限管理 | 自定义本组织角色（如仓库管理员、采购员、库管员） |
| 业务管控 | 审核单据、处理异常、审批盘点 |
| 日志管理 | 查看本组织所有操作日志 |
| 数据管理 | 本组织数据备份恢复、批量导入导出 |

### 2.4 组织员工（可自定义角色）

| 角色 | 职责 |
|------|------|
| 仓库管理员 | 仓库/货位基础管理、库存预警处理、盘点审核 |
| 库管员 | 执行出入库操作、录入库存变动、上报异常 |
| 采购员 | 发起入库申请、关联供应商、提交采购单据 |
| 统计员 | 查看/导出各类报表、统计库存周转率 |

> 角色可灵活自定义，一个员工可分配多个角色（权限取并集）

## 三、功能模块

```

│   ├── 账号管理（CRUD、密码重置、禁用/启用）
│   └── 角色与权限（自定义角色、权限分配）
├── 商品管理
│   ├── 商品信息（CRUD、批量操作）
│   ├── 商品分类（多级分类）
│   └── 商品标签（拓展）
├── 供应商管理

│   └── 供应商关联（商品-供应商对应关系、供货统计）
├── 出入库管理
│   ├── 入库管理（采购/退货/调拨/盘盈入库）
```

### 3.2 多组织管理

#### 组织管理
- **创建**：录入组织名称、联系人、电话、地址、备注 → 生成唯一组织ID
- **编辑**：修改基础信息（禁用状态除外）
- **禁用/启用**：禁用后用户无法登录，数据保留；可重新启用
- **删除**：仅无业务数据的组织可删除，有数据的仅可禁用
- **列表**：支持按名称、状态搜索，可导出

#### 组织切换（仅系统管理员）
系统管理员可切换至任意组织查看基础配置和业务数据（仅查看，不可操作）

---

### 3.3 账号与权限

#### 账号管理
- **创建**：录入账号、密码（初始密码可重置）、姓名、联系方式、角色
- **编辑**：修改基础信息（账号名称不可改）
- **密码**：自行修改（需原密码）/ 管理员重置（重置为初始密码）
- **禁用/启用**：禁用后无法登录
- **列表**：按账号名称、角色、状态搜索，可导出

#### 角色与权限
- **角色创建**：输入角色名称、备注、勾选操作权限
- **角色编辑**：修改后立即生效，影响所有关联账号
- **角色删除**：仅未关联员工的角色可删除
- **权限分配**：一个员工可分配多个角色（权限取并集）

---

### 3.4 商品管理

#### 商品信息
- **新增**：名称、编码（唯一）、分类、规格、单位、单价、成本价、产地、图片等，支持批量新增
- **编辑**：修改除编码外的信息，同步更新关联单据和库存
- **禁用/启用**：禁用后不可出入库，库存保留；可重新启用
- **删除**：仅无出入库记录、无库存的商品可删除
- **列表**：按名称/编码/分类/状态搜索，按单价/库存排序，可导出

#### 商品分类
- 支持多级分类（如：电子产品 → 手机/电脑）
- 有商品关联的分类不可删除

#### 商品标签（拓展）
- 创建标签（名称、颜色、描述）
- 为商品关联标签，便于筛选

---

### 3.5 供应商管理

#### 供应商信息
- **新增**：名称、编码（唯一）、联系人、电话、邮箱、地址、合作状态等，支持批量新增
- **编辑**：同步更新关联入库单据和统计数据
- **禁用/启用**：禁用后不可关联新单据，已关联单据不受影响
- **删除**：仅未关联入库单据的供应商可删除
- **列表**：按名称/编码/合作状态搜索，可导出，可查看关联入库记录

#### 供应商关联
- **商品-供应商关联**：一个商品可关联多个供应商，记录单价、供货周期
- **入库关联供应商**：采购/退货入库时自动带出默认单价，便于统计供货情况
- **供货统计**：统计每个供应商的供货总量、金额、商品种类

---

### 3.6 出入库管理

> 所有单据流程：**申请 → 审核 → 执行 → 已完成 → 归档**  
> 关联操作人、操作时间，支持异常处理

#### 入库管理

| 入库类型 | 说明 | 必须关联供应商 |
|----------|------|----------------|
| 采购入库 | 商品采购入库 | ✅ |
| 退货入库 | 客户退货入库 | ✅ |
| 调拨入库 | 其他仓库调入 | ❌ |
| 盘盈入库 | 盘点差异入库 | ❌ |
| 其他入库 | 自定义类型 | ❌ |

**流程**：
1. 有权限员工发起申请（选择类型/仓库/货位/商品/供应商 → 录入数量/单价/日期）
2. 有权限员工审核（通过/驳回）
3. 库管员执行入库（确认实际数量/货位/时间）→ 库存自动增加
4. 单据归档（归档后不可修改）

**异常处理**：数量不符/商品损坏 → 发起异常申请 → 审核 → 调整库存和单据

#### 出库管理

| 出库类型 | 说明 |
|----------|------|
| 销售出库 | 商品销售出库 |
| 退货出库 | 退给供应商出库 |
| 调拨出库 | 调至其他仓库 |
| 盘亏出库 | 盘点差异出库 |
| 其他出库 | 自定义类型 |

**流程**：同上（入库流程对称）

#### 出入库记录查询
按商品/仓库/类型/时间/操作人查询，显示时间/数量/单价/单据编号/供应商（仅入库）

---

### 3.7 库存管理

#### 实时库存查询
- 显示：商品名称/编码/规格/单位/库存数量/可用库存/锁定库存/仓库/货位/成本价/库存金额
- 支持按商品/仓库/货位搜索，按库存/金额排序
- 查看商品库存明细（所有出入库记录、变动历史）

#### 库存预警
- **设置**：按单个商品或按分类批量设置最低/最高库存阈值
- **提醒**：触发预警时首页显示 + 向指定角色发送系统消息
- **处理**：查看预警列表 → 点击商品查看详情 → 发起采购入库或出库调整

#### 库存调拨
- 发起调拨申请（选择调出/调入仓库 → 添加商品/数量/日期）
- 审核通过后执行
- 调出仓库减库存，调入仓库加库存，记录调拨记录

#### 库存盘点
```
盘点计划 → 审核 → 执行（录入实际数量）→ 生成差异（盘盈/盘亏）
    ↓
差异处理 → 审核 → 系统自动调整库存 → 生成单据 → 完成盘点
```

---

### 3.8 仓库与货位管理

#### 仓库管理
- **新增**：名称、编码（唯一）、地址、负责人、联系方式、仓库类型（普通/冷藏等）
- **编辑/删除**：删除需仓库无库存、无出入库记录
- **列表**：按名称/编码搜索，可导出

#### 货位管理
- 支持多级：仓库 → 区域 → 货架 → 货位
- 录入编码（唯一）、名称、所属仓库、区域
- 显示货位状态（空闲/占用）、占用商品、库存数量

---

### 3.9 报表管理

| 报表类型 | 内容 |
|----------|------|
| 库存报表 | 库存明细/汇总/变动趋势，支持按时间/仓库/分类筛选，统计周转率/金额 |
| 出入库报表 | 汇总/明细/趋势，支持按类型/时间/商品/仓库筛选，统计数量/金额 |
| 供应商报表 | 供货汇总/明细/趋势，支持按时间/供应商筛选，统计供货占比/频次 |
| 盘点报表 | 盘点计划汇总/差异明细，按时间/仓库筛选，统计准确率 |
| 自定义报表 | 组织管理员自定义统计维度/筛选条件 |

> 所有报表均按组织隔离，支持导出（Excel/PDF）和打印

---

### 3.10 日志管理

| 日志类型 | 记录内容 | 查看权限 |
|----------|----------|----------|
| 操作日志 | 操作人/时间/IP/模块/内容/结果 | 系统管理员（全局）/ 组织管理员（本组织） |
| 系统日志 | 运行状态/异常信息/登录失败 | 仅系统管理员 |
| 数据日志 | 商品/库存/供应商等的修改前后数据 | 组织管理员（本组织） |

- 日志保留周期可配置
- 支持按操作人/模块/时间/结果搜索，可导出

---

### 3.11 系统配置

#### 系统级配置（系统管理员）
- 全局参数：备份周期、密码规则、日志保留周期、预警默认值、单据编号规则
- 全局字典：出入库类型、商品单位、仓库类型等（适用于所有组织）
- 备份与恢复：设置自动备份计划、手动备份、恢复数据

#### 组织级配置（组织管理员）
- 组织参数：库存预警阈值、单据审核流程、出入库规则
- 组织字典：在系统级字典基础上新增自定义项（仅本组织适用）
- 打印配置：自定义单据/报表的打印模板

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 单操作响应时间 | ≤1秒 |
| 列表查询（≤1000条） | ≤2秒 |
| 大数据量查询（＞1000条） | ≤5秒 |
| 同时在线用户 | ≥100人 |
| 同时并发操作 | ≥50 |
| 单组织商品容量 | ≥10,000条 |
| 单组织单据容量 | ≥100,000条 |

### 4.2 安全性需求

| 方面 | 要求 |
|------|------|
| 数据隔离 | 各组织数据完全隔离 |
| 存储安全 | 数据库加密，敏感数据（密码/开户行）加密处理 |
| 权限安全 | 严格权限管控，操作日志可追溯，异常操作及时提醒 |
| 登录安全 | 密码加密存储，复杂度校验，5次失败锁定账号，支持IP限制 |
| 备份安全 | 自动定期备份，备份文件加密，支持手动备份恢复 |
| 防护能力 | 防SQL注入、XSS跨站脚本攻击 |

### 4.3 可靠性需求

| 指标 | 要求 |
|------|------|
| 连续运行 | ≥72小时无异常，无崩溃/死锁 |
| 恢复时间 | 异常后≤30分钟恢复 |
| 数据准确性 | 录入/修改/删除准确，库存实时同步，单据与库存一致 |
| 容错性 | 错误提示明确，不导致崩溃，未审核单据可撤销 |

### 4.4 易用性需求

- 界面简洁专业，布局合理，操作流程清晰
- 支持批量操作，常用功能快捷入口
- 提示信息明确（成功/错误/必填项）
- 内置帮助文档，常见问题可查

### 4.5 可扩展性需求

- **功能拓展**：新增模块（ERP对接/WMS对接/移动端）无需大规模修改核心代码
- **数据拓展**：支持自定义商品/供应商/单据字段
- **接口拓展**：提供标准API，支持与财务/采购等系统对接

---

## 五、数据需求

### 5.1 数据分类

**系统级数据**
- 组织信息、系统管理员账号、系统配置、全局字典、系统日志

**组织级数据**
- 员工账号、角色权限、商品信息、供应商信息、仓库货位、库存数据、出入库单据、盘点记录、组织配置、组织日志

### 5.2 存储要求

| 项目 | 要求 |
|------|------|
| 数据库 | 关系型数据库（MySQL/PostgreSQL），支持数据分区 |
| 核心业务数据 | 长期存储 |
| 操作日志 | 保留≥1年 |
| 系统日志 | 保留≥6个月 |
| 自动备份 | 周期≤1天 |
| 备份文件保留 | ≥7天 |

### 5.3 数据交互

| 功能 | 说明 |
|------|------|
| 导入 | Excel格式，导入前验证，失败提示错误行和原因 |
| 导出 | Excel/PDF格式，可自定义导出字段，命名规范（如`商品列表_20260409.xlsx`） |
| 同步 | 出入库实时同步库存，单据状态实时更新，修改实时记录数据日志 |

## 七、验收标准

### 7.1 功能验收
- [ ] 核心功能（多组织管理、账号权限、商品管理、供应商管理、出入库管理、库存管理）正常运行
- [ ] 多组织数据完全隔离，无越权访问
- [ ] 三级权限架构生效，无越权操作
- [ ] 出入库流程完整（申请→审核→执行→异常处理）
- [ ] 库存变动准确，报表数据与业务一致
